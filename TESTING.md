# Gringotts Budget Vault Testing

## Data boundary

All automated tests use fictional fixtures and isolated browser contexts.

Tests must never read, commit, upload, or publish a real household vault, bank export, statement, report, screenshot, account identifier, or generated financial artifact.

Synthetic sources include:

- `tests/fixtures/synthetic-vault.json`;
- signed CSV;
- separate debit/credit CSV;
- QFX/OFX-family data;
- fictional legacy Gringotts JSON packages;
- deterministic malformed-input mutations.

## Requirements

- Node.js 24 or newer;
- Python 3 for the local static server;
- Git history for privacy scanning.

## Browser-free parser and syntax gate

Run first:

```bash
npm run test:parser
```

Node's built-in test runner covers format detection, delimiters, quoting, field mapping, date and amount validation, OFX-family parsing, limits, and deterministic malformed-input mutations.

GitHub also runs `node --check` against the current v115 import modules and v116 boot, presentation, and reporting modules before installing a browser.

## Local browser setup

```bash
npm ci --ignore-scripts
npx playwright install chromium
```

Fast release-candidate checks:

```bash
npm run test:preflight
```

## v116 architecture coverage

### Boot and primary navigation

Tests verify:

- v116 boots without module errors;
- `Mischief Managed. Money Managed` remains visible;
- Dashboard, Money, Calendar, Reports, Activity, and Tools remain the six primary destinations;
- navigation and presentation controls do not change the vault;
- one v116 boot entry loads the established runtime plus v115 features and the v116 presentation layer;
- the stable v105 rescue page remains linked;
- no service worker is registered;
- normal navigation makes no network write request.

### Report preview architecture

Tests verify:

- the native report-page select contains eight pages;
- only one report page is visible on screen;
- Previous and Next update the selected page;
- range controls and downloads remain available;
- custom ranges and prior-year comparisons still persist locally;
- all eight report pages are visible in print media;
- the 33-sheet workbook, meeting pack, Guided Plan, CSV, and annual tracker remain available;
- v116 filenames are generated by the capture-phase export override;
- every report page receives axe coverage;
- desktop and phone report layouts remain inside the viewport.

### Import and restore architecture

Tests verify:

- Import transactions is the initial task;
- full restore is hidden until explicitly selected;
- switching tasks performs no data write;
- Inspect, Map, and Reconcile progress reflects session state;
- the v115 parser, mapping, duplicate, backup, rollback, and verification engine remains unchanged;
- legacy v109 JSON import scenarios continue through the current UI;
- empty restore remains blocked after switching to Full vault restore;
- restore destination remains `gringottsBudgetVault.latest`;
- both task surfaces receive axe coverage;
- phone, tablet, and desktop layouts remain contained.

### Responsive navigation

Tests verify:

- Activity keeps all five secondary destinations;
- narrow-phone Activity navigation scrolls horizontally rather than stacking oversized controls;
- touch controls retain usable height;
- no primary page creates horizontal document overflow.

### Observer stability

The Reports and Import / Restore enhancement observer must settle after initial rendering and after explicit page or task changes. Continuous child-list rewrites fail the stability tests.

## Accessibility and visual quality

The quality suite blocks serious or critical axe findings for configured WCAG and best-practice tags.

Keyboard coverage includes:

- Skip to content;
- visible focus;
- secondary tab semantics and arrow navigation;
- accessible table regions;
- mobile menu Escape behavior;
- unique rendered IDs;
- native report and mapping selects.

Privacy-safe visual contracts cover:

- Dashboard desktop;
- Reports desktop and phone with only the summary page visible initially;
- Import phone with bank import visible and restore hidden;
- Activity phone with compact horizontal secondary navigation;
- range-control columns, main width, topbar placement, control height, and horizontal overflow.

No PNG baseline is committed. Screenshots, traces, video, axe JSON, Playwright reports, and Lighthouse files upload only on failure.

## Staged GitHub Actions

Draft pull requests skip protected jobs.

When marked ready:

1. Parser and static preflight checks syntax and pure parser behavior.
2. Desktop installs Chromium and runs it before Firefox/WebKit installation.
3. Responsive installs Chromium and runs Android before iPad/iPhone WebKit installation.
4. Quality runs keyboard and visual contracts before axe.
5. Lighthouse runs independently in parallel.
6. Privacy, supply-chain, and CodeQL jobs run in parallel.

Concurrency cancellation stops superseded runs. Diagnostics upload only on failure.

## Security gates

The final candidate runs:

- full-history financial-data path and identifier scanning;
- full-history Gitleaks;
- Dependency Review;
- locked `npm audit --audit-level=high` with lifecycle scripts disabled;
- CodeQL with `security-extended` queries;
- repository drift checks for action pinning, permissions, headers, parser purity, guarded writes, v116 no-storage/no-network presentation, staged installs, and required files.
